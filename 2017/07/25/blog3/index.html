学习gulp(一)

gulp 和 grunt 非常类似，但相比于 grunt 的频繁 IO 操作，gulp 的流操作，能更快地更便捷地完成构建工作。所以现在大多数项目都采用的gulp构建工具，接下来跟我一起学习一下这个讨喜的小东西吧！

简单介绍一下：

gulp是前端开发过程中对代码进行构建的工具，是自动化项目的构建利器；不仅能对网站资源进行优化，而且在开发过程中很多重复的任务能够使用正确的工具自动完成；使用它，我们不仅可以很愉快的编写代码，而且大大提高我们的工作效率呢。

准备工作：

使用gulp这么前卫的工具，当然少不了我们的nodejs啦，这么有名应该都知道的吧，但是不了解的童鞋可以先去了解一下，这里我们大概知道怎么用就可以了，我们要先从官网下载安装它，然后进入命令行时代！（我个人用的是gitbash，我接下来的命令行都是在这上面运行的，大家也可以用windows自带的命令行终端）

通过nodejs的npm全局安装和项目安装gulp，其次在项目里安装所需要的gulp插件，然后新建gulp的配置文件gulpfile.js并写好配置信息（定义gulp任务），最后通过命令提示符运行gulp任务即可。

新建package.json文件：

package.json是基于nodejs项目必不可少的配置文件，它是存放在项目根目录的普通json文件；

这些参数根据其中文意思大致知道是什么意思，现在我们注意到其中一个参数devDependencies这个是项目依赖的插件，而其中的gulp-less我们没有装，现在，如果你是手动建的文件夹，并键入的文本，可在这项版本号中写入空字符串（因为我们忘了安装插件gulp-less)，然后先保存这个文件，运行命令：npm install gulp-less –save-dev 嗯哼！这个时候空字符串里自动就填写了该插件的版本号。惊不惊喜，意不意外？谁让这是必不可少的配置文件呢！后面的插件就类似这样进行安装就好（注意：这是本地安装哦！）。
为了能正常使用，我们还得本地安装gulp：npm install gulp –save-dev；
PS：细心的你可能会发现，我们全局安装了gulp，项目也安装了gulp，全局安装gulp是为了执行gulp任务，本地安装gulp则是为了调用gulp插件的功能。
